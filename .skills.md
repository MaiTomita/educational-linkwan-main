## Skill: Google Authentication Flow

### When to use
- Implementing or debugging Google Sign-In
- Fixing OAuth errors
- Understanding the authentication flow

### Implementation Pattern

**Frontend (client/src/shared/googleIdentity.js):**
```javascript
// Initialize Google Identity Services
google.accounts.id.initialize({
  client_id: GOOGLE_CLIENT_ID,
  callback: handleCredentialResponse
});

// On callback, send credential to backend
async function handleCredentialResponse(response) {
  const { credential } = response;
  const result = await authWithGoogleCredential(credential);
  // On success, redirect to Setting.html
}
```

**Backend (server/src/index.js):**
```javascript
// Verify ID token and create session
app.post('/api/auth/google', async (req, res) => {
  const { credential } = req.body;
  const user = await verifyGoogleIdToken(credential);
  await upsertUserAndAppendLogin({ user, ip, userAgent });
  const token = signSession(user);
  setSessionCookie(res, token);
  return res.json({ ok: true, user });
});
```

### Common Issues
- **Origin not authorized**: Add origin to Google Cloud Console
- **Invalid ID token**: Check CLIENT_ID matches between frontend/backend
- **Cookie not set**: Verify CORS credentials and CLIENT_ORIGIN

## Skill: Google Sheets Database Operations

### When to use
- Adding new data to sheets
- Querying user information
- Implementing new sheet-based features

### Available Functions (server/src/sheets.js)

**Pattern for Reading:**
```javascript
const response = await sheets.spreadsheets.values.get({
  spreadsheetId: SPREADSHEET_ID,
  range: 'users!A:Z'
});
const rows = response.data.values;
```

**Pattern for Writing:**
```javascript
await sheets.spreadsheets.values.append({
  spreadsheetId: SPREADSHEET_ID,
  range: 'users!A:Z',
  valueInputOption: 'USER_ENTERED',
  resource: { values: [[email, name, picture, sub, timestamp]] }
});
```

**Pattern for Updating:**
```javascript
await sheets.spreadsheets.values.update({
  spreadsheetId: SPREADSHEET_ID,
  range: `users!F${rowIndex}`,
  valueInputOption: 'USER_ENTERED',
  resource: { values: [[newValue]] }
});
```

### Sheet Structure
- **users**: `email | name | picture | sub | first_login | last_login`
- **login_history**: `timestamp | email | ip | user_agent`

### Best Practices
- Always use try/catch for Sheets API calls
- Validate data before writing to sheets
- Use service account for server-side operations
- Headers are in row 1, data starts from row 2

## Skill: Adding Protected API Routes

### When to use
- Creating new authenticated endpoints
- Implementing user-specific features
- Accessing current user data

### Implementation Steps

1. **Add route with middleware:**
```javascript
app.get('/api/your-endpoint', requireAuth, async (req, res) => {
  // req.user is available (from JWT)
  const currentUser = req.user;
  // ... your logic
});
```

2. **Validate request body:**
```javascript
import { z } from 'zod';

const schema = z.object({
  field: z.string(),
  // ... other validations
});

const data = schema.parse(req.body);
```

3. **Return consistent format:**
```javascript
// Success
return res.json({ ok: true, data: result });

// Error
return res.status(400).json({ ok: false, error: message });
```

4. **Add client function:**
```javascript
// client/src/shared/api.js
export async function yourEndpoint(params) {
  return request('/api/your-endpoint', {
    method: 'POST',
    body: params
  });
}
```


## Skill: Debugging Common Issues

### CORS Errors
**Symptoms:** Network errors, blocked requests, "CORS policy" in console

**Solutions:**
1. Check `CLIENT_ORIGIN` in `server/.env` matches frontend URL
2. Ensure `credentials: true` in server CORS config
3. Verify `credentials: 'include'` in all fetch requests

### Session/Cookie Issues
**Symptoms:** User logged out immediately, `/api/me` returns 401

**Solutions:**
1. Check cookie is being set (DevTools → Application → Cookies)
2. Verify JWT_SECRET is set in server `.env`
3. Ensure same domain for frontend/backend (localhost)
4. Check `httpOnly`, `secure`, `sameSite` cookie attributes

### Google Sheets Permission Errors
**Symptoms:** "The caller does not have permission"

**Solutions:**
1. Share spreadsheet with service account email
2. Verify service account has Google Sheets API enabled
3. Check `GOOGLE_SHEETS_SPREADSHEET_ID` is correct
4. Ensure credentials JSON is valid

### Build/Import Errors
**Symptoms:** Module not found, import errors

**Solutions:**
1. Check file extensions (.js, .jsx) are correct
2. Verify paths use `/` not `\`
3. Ensure `type: "module"` in package.json
4. Run `npm install` after pulling changes

## Workflow: Complete Feature Implementation

### Example: Adding User Profile Update

1. **Add Sheets column:**
   - Manually add column to `users` sheet (e.g., `profile_data`)

2. **Create API endpoint:**
```javascript
// server/src/index.js
app.put('/api/profile', requireAuth, async (req, res) => {
  const schema = z.object({ profile: z.string() });
  const { profile } = schema.parse(req.body);
  
  // Update in sheets (implement in sheets.js)
  await updateUserProfile(req.user.email, profile);
  
  return res.json({ ok: true });
});
```

3. **Implement Sheets function:**
```javascript
// server/src/sheets.js
export async function updateUserProfile(email, profile) {
  // Get all users, find row, update cell
  // ...implementation
}
```

4. **Add client API:**
```javascript
// client/src/shared/api.js
export async function updateProfile(profile) {
  return request('/api/profile', {
    method: 'PUT',
    body: { profile }
  });
}
```

5. **Use in component:**
```javascript

  await updateProfile(profileData);
  alert('保存しました');

```

## Code Style Preferences

### React Components
- Use functional components with hooks
- Destructure props
- One component per file
- CSS imports at top

### Error Handling
- Always use try/catch for async operations
- Log errors for debugging
- Return user-friendly error messages
- Don't expose sensitive info in errors

### Comments
- Use Japanese comments for business logic explanation
- Use English comments for technical implementation details
- Document complex algorithms
- Explain "why" not "what"

### Testing (Manual)
1. Test happy path
2. Test error cases (invalid input, network failures)
3. Test authentication (logged in/out states)
4. Verify console has no errors
5. Check mobile responsiveness

## Project-Specific Patterns to Follow

1. **Always use the shared API wrapper** (`client/src/shared/api.js`)
2. **All protected routes must use `requireAuth` middleware**
3. **All Sheets operations must go through `sheets.js`**
4. **Google authentication must use `googleIdentity.js`**
5. **Environment variables must be validated in `env.js`**
6. **Keep credentials out of git** (use `.gitignore`)
7. **Maintain multi-language support** in documentation
8. **Preserve legacy backup** for reference (`legacy_static_backup/`)
