# Educational LinkWAN - Copilot Instructions

## プロジェクト概要 / Project Overview
Educational LinkWANは、Google Sheets をデータベースとして使用する教育向けのMERNスタック風アプリケーションです。

**主な技術スタック:**
- **Frontend**: Vite + React (Multi-page application)
- **Backend**: Node.js + Express
- **Database**: Google Sheets API (Service Account)
- **Authentication**: Google Identity Services (OAuth2) + JWT session cookies
- **Language**: Multi-lingual documentation (Japanese/English)

## ディレクトリ構造 / Directory Structure

```
/client/                 # Vite + React frontend
  /src/title/           # Title page components
  /src/setting/         # Setting page components  
  /src/shared/          # Shared utilities (api.js, googleIdentity.js)
  /public/              # Static assets
  Title.html            # Entry point for Title page
  Setting.html          # Entry point for Setting page

/server/                # Express backend
  /src/
    index.js           # Main server & routes
    auth.js            # JWT session & middleware
    google.js          # Google OAuth verification
    sheets.js          # Google Sheets operations
    env.js             # Environment validation (Zod)
  /credentials/        # Google Cloud credentials (NOT in git)

/legacy_static_backup/  # Legacy HTML/CSS/JS (reference only)
```

## コーディング規則 / Coding Conventions

### General
- Use **ES modules** (`import/export`) throughout the project
- Prefer `async/await` over promise chains
- Use `const` by default, `let` when mutation is needed
- Keep functions small and focused on single responsibility

### Backend (server/)
- Use **Zod** for request validation
- All API endpoints should return JSON with `{ ok: boolean, ... }` structure
- Error handling: catch errors and return appropriate HTTP status codes
- Authentication: use `requireAuth` middleware for protected routes
- Environment variables: validate with `env.js` using Zod schemas

### Frontend (client/)
- React components use **.jsx** extension
- CSS modules or separate .css files for styling
- API calls go through `src/shared/api.js`
- All fetch requests include `credentials: 'include'` for cookies
- Google Sign-In uses `src/shared/googleIdentity.js`

### Environment Variables
**Backend (.env):**
- `GOOGLE_CLIENT_ID` - OAuth 2.0 Client ID
- `GOOGLE_SERVICE_ACCOUNT_JSON` - Service account credentials (JSON string or file path)
- `GOOGLE_SHEETS_SPREADSHEET_ID` - Target spreadsheet ID
- `PORT` - Server port (default: 4000)
- `CLIENT_ORIGIN` - CORS allowed origin (default: http://localhost:5173)
- `JWT_SECRET` - JWT signing secret

**Frontend (.env):**
- `VITE_GOOGLE_CLIENT_ID` - OAuth 2.0 Client ID (must match backend)
- `VITE_API_BASE` - Backend API URL (default: http://localhost:4000)

## API エンドポイント / API Endpoints

### Public
- `GET /health` - Health check
- `POST /api/auth/google` - Authenticate with Google ID token
  - Body: `{ credential: string }`
  - Returns: `{ ok: true, user: {...} }` + sets session cookie

### Protected (requires authentication)
- `GET /api/me` - Get current user info
- `POST /api/logout` - Clear session cookie

## 重要な注意事項 / Important Notes

1. **Google Cloud Setup Required:**
   - OAuth 2.0 Client ID with authorized JavaScript origins
   - Service Account with Google Sheets API access
   - Spreadsheet shared with service account email

2. **CORS Configuration:**
   - Backend CORS must allow frontend origin
   - Credentials must be included in all API requests

3. **Session Management:**
   - JWT tokens stored in HTTP-only cookies
   - Cookies named `session`
   - Logout clears the session cookie

4. **Multi-page Navigation:**
   - `Title.html` → Login page
   - `Setting.html` → Protected user dashboard
   - Navigation happens via `window.location.href`

## Common Tasks

### Adding New API Endpoint
1. Define route in `server/src/index.js`
2. Create request schema with Zod
3. Add middleware (`requireAuth` if protected)
4. Implement handler with try/catch
5. Add client-side function in `client/src/shared/api.js`

### Adding New Page
1. Create HTML file in `client/` (e.g., `NewPage.html`)
2. Create component folder in `client/src/newpage/`
3. Add entry point in `client/vite.config.js` build input
4. Implement React components and styles

### Google Sheets Operations
- All operations in `server/src/sheets.js`
- Use `googleapis` library
- Service account authentication handled in `google.js`
- Sheet names: `users`, `login_history`

## トラブルシューティング / Troubleshooting

**"no registered origin" error:**
→ Add `http://localhost:5173` to Google Cloud Console OAuth authorized origins

**Sheets API permission denied:**
→ Share spreadsheet with service account email address

**Login redirects fail:**
→ Verify `GOOGLE_CLIENT_ID` matches in both client and server .env files

**Cookie not set:**
→ Check `CLIENT_ORIGIN` in server .env matches frontend URL
→ Ensure `credentials: 'include'` in fetch requests
